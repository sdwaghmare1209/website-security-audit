# Website Security Audit Portfolio

This repository contains a complete website security audit performed in a controlled lab environment. The purpose of this project is to demonstrate practical web application security assessment skills, professional reporting, and vulnerability analysis using industry-standard methodologies.

⚠️ Disclaimer:
All testing was conducted on deliberately vulnerable applications in an isolated lab environment. No real-world systems were tested.

## Project Overview

- Assessment Type: Website & Web Application Security Audit
- Target Environment: Metasploitable2 (Lab VM)
- Web Application: Damn Vulnerable Web Application (DVWA)
- Testing Machine: Kali Linux
- Methodology: OWASP Web Security Testing Guide (WSTG)

## Vulnerabilities Identified

- Default Credentials (Critical)
- OS Command Injection (Critical)
- SQL Injection (High)
- Blind SQL Injection (High)
- Insecure File Upload – Remote Code Execution (Critical)
- Reflected Cross-Site Scripting (Medium)
- Stored Cross-Site Scripting (High)

- Individual vulnerability findings with proof-of-concept
- Professional audit documentation
- Evidence collected during testing
